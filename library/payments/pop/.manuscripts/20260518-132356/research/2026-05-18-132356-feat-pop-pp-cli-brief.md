# POP Printing Press brief

- Scope: publish POP as a `payments` catalog entry with an intentionally frozen 8-tool surface.
- Positioning: Europe-first electronic invoicing for compliant XML, PEPPOL, PDF, validation, status tracking, and archival workflows.
- Sources used: POP public product pages, POP API behavior observed against staging, and the POP MCP README as an operation-level reference.
- Auth contract: `POP_API_KEY` only.
- Environment note: smoke validation was run against POP staging via `POP_BASE_URL=https://staging7.popapi.io/wp-json/api/v2`.
